# Файл для агента, в котором он при необходимости формулирует и хранит для себя понятные ему заметки, промпты и инструкции на основе опыта взаимодействия с пользователем по типовым задачам. Обновляет, уточняет и использует их в зависимости от кейса.

## Encoding rules for `.md` files

Все `.md`-файлы в vault (включая создаваемые и редактируемые) должны быть в чистoй **UTF-8 без BOM** (собственно, как их создаёт Obsidian). Никогда не допускай двойного кодирования или mojibake.

- При чтении `.md`-файлов всегда используй `encoding='utf-8'`.
- При записи — тоже `encoding='utf-8'`.
- Проверяй, что текстовый редактор/библиотека (например, PowerShell, Python) действительно работают с UTF-8, а не с системной кодировкой (Windows-1251).

### Если всё же произошла двойная кодировка (mojibake)

Симптом: текст содержит кракозябры вида `Р“РћРЎРў` вместо `ГОСТ`, `Р‘РёР±Р»РёРѕРіСЂР°С„РёСЏ` вместо `Библиография`.

Типичная причина: исходный UTF-8 был прочитан как Windows-1251/CP1251, а затем результат сохранён как UTF-8. Порядок исправления:

1. Прочитать файл как UTF-8 (получаем строку с CP1251-кракозябрами).
2. Закодировать строку как CP1251 (получаем исходные UTF-8 байты).
3. Декодировать байты как UTF-8 (получаем чистый текст).
4. Сохранить результат как UTF-8.

Если какой-то байт отсутствует в CP1251 (например 0x98), отображать его напрямую как байт-в-байт (Latin-1 fallback) — для данного vault это корректно восстанавливает оригинальный UTF-8.

## Rule: Subprocess extraction — полный вынос, минимум дубляжа

При создании подпроцесса (страницы-подпроцесса по AGENTS.md 3.3):

1. **Выноси всё, оставляй ссылки.** Всю детальную информацию (пошаговые инструкции, таблицы, чек-листы, нормативные ссылки) помещай в подпроцесс. На странице-доноре оставляй только общую смысловую информацию (1–3 предложения + ссылка `→ [[Подпроцесс]]`).

2. **Проверь всех доноров.** Прежде чем создавать подпроцесс, grep'ом найди все страницы, где может быть релевантная информация. Если подпроцесс применим в нескольких процессах — заимствуй информацию из всех таких страниц, а на каждой из них оставь ссылку на подпроцесс.

3. **Проверь, нет ли уже такого подпроцесса.** Прежде чем создавать новый, убедись, что не существует аналогичной страницы под другим именем. Если существует — дополни её, а не создавай дубликат.

4. **Проверь на orphan.** После создания убедись, что хотя бы одна другая страница ссылается на новый подпроцесс (иначе он станет orphan). Если не ссылается — добавь `[[wikilink]]` на подпроцесс во все страницы-доноры.

5. **Удали дубликаты с доноров.** После выноса информации проверь страницы-доноры на оставшиеся дублирующиеся утверждения. Если какое-либо положение теперь полностью покрыто подпроцессом — удали его с донора и замени ссылкой.

## Rule: Wikilink all bare references

При редактировании или linting wiki-страниц ищи текстовые ссылки на источники, разделы и страницы без `[[wikilinks]]`. Примеры: «СП 63.13330», «п. 53», «ст. 67, 68, 71, 162», «п. 2 ст. 3», «по ГОСТ Р 8.736-2011», «п. 4.5–4.8», «см. Типовые вопросы эксперту», «Постановление ВАС №23», «Федеральный закон № 73-ФЗ», «Приказ Минюста №237», «МДС 81-35.2004».

Правила обработки:

1. **Ссылка на пункт/раздел источника** → убедись, что на wiki-странице документа есть дословная цитата этого пункта из `raw/`. Если цитаты нет — добавь её (прочитай соответствующий `raw/`-файл, выбери нужный пункт, вставь как блок цитаты с указанием пункта). Затем проставь `[[wikilink#пункт]]` на точную строку с цитатой.

2. **Ссылка на источник в целом** (ГОСТ, СП, ФЗ, Постановление и т.п.) → замени на `[[wikilink]]` на страницу документа (если страница существует). Если страницы нет — создай по шаблону 3.4 AGENTS.md.

3. **Ссылка на раздел wiki-страницы** («см. Типовые вопросы эксперту», «см. раздел Права эксперта») → замени на `[[wikilink#Раздел]]`.

4. **Источника нет в `raw/`** → оставь как есть, текстом. Не создавай страницы без raw-источника.

5. **Цепочка пунктов** («ст. 67, 68, 71, 162») → проставь `[[wikilink#пункт]]` на каждый отдельно, если есть соответствующие страницы. Если на одной странице — можно общую ссылку на страницу с уточнением в тексте.

6. **Отображаемый текст**  → по общим правилам раздела 5 AGENTS.md. 
- Ссылка на конкретный пункт: `[[документ#пункт|пункт Х.Х. документ]]`. 
- Если места мало (пример - таблицы, цепочки, диапазоны пунктов «п. 4.5–4.8»), сокращай до пункта `[[документ#пункт|пункт Х.Х.]]`. После цепочек и диапазонов оставь `[[wikilink]]` на страницу документа в целом.
