# Obsidian LLM Wiki

Этот репозиторий хранит Obsidian vault для ведения локальной LLM-ориентированной базы знаний.

Начинайте с `wiki/index.md`, затем переходите по связанным тематическим страницам. Исходные или слабо обработанные материалы находятся в `raw/`.

## Структура

- `wiki/` - обработанные заметки базы знаний и индексные страницы.
- `raw/` - исходные или слабо обработанные заметки-источники.
- `AGENTS.md` - schema-файл с правилами для LLM-агента, который поддерживает vault.
- `.obsidian/` - общие настройки Obsidian для этого vault.
- `.gitignore` - исключает локальное состояние рабочего пространства, кэши, системные и временные файлы.

## Что отслеживать в Git

Коммитьте заметки, исходные материалы, вложения и осознанно выбранные общие настройки Obsidian.

Не коммитьте локальное состояние рабочего пространства, кэши, временные файлы и настройки плагинов, если они содержат API-ключи, токены, учетные данные или приватные данные сервисов.

Перед коммитом настроек community-плагинов проверяйте файлы вроде `.obsidian/plugins/*/data.json` на наличие секретов.

## Работа c Vault

Откройте эту папку как Obsidian vault. Используйте `wiki/index.md` как основную точку входа для навигации, а `raw/` - когда нужно проверить или пересмотреть исходные материалы.

## Команды агенту

Wiki не обновляется в фоне сама по себе: агент действует в чате Cursor по вашему запросу или по явной задаче в сессии. Ниже — типовые операции и что попросить.

| Операция            | Когда                                               | Примеры запроса                                                           | Что делает агент                                                                                                            |
| ------------------- | --------------------------------------------------- | ------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **Ingest**          | Новый файл в `raw/` или нужно «переварить» источник | «Ingest `raw/...`», «Обработай новые файлы в raw», «Что ещё не ingested?» | Саммари, страницы в `wiki/`, wikilinks, обновление `wiki/Source Notes.md`, `wiki/log.md`, при необходимости `wiki/index.md` |
| **Query**           | Вопрос по базе                                      | «Что такое …?», «Как у нас описано …?»                                    | Сначала `wiki/`, при нехватке — `raw/`; ответ с цитатами; wiki не переписывает молча                                        |
| **Crystallization** | Устойчивый вывод из ответа стоит сохранить          | «Сохрани это в wiki», «Создай страницу про …»                             | Новая или обновлённая страница в `wiki/` (по согласию, не по умолчанию)                                                     |
| **Сверка с `raw/`** | Пополнение или удаление в `raw/`                    | «Что нового в raw?», «Согласуй wiki с raw»                                | Сравнение с `wiki/Source Notes.md`; для удалённых — пометки, понижение confidence, запись в log                             |
| **Lint**            | Проверка здоровья базы                              | «Проверь wiki», «Lint»                                                    | Битые wikilinks, дубли, orphans, устаревшие утверждения; отчёт, содержательные правки — по договорённости                   |

Практический ритм: capture в `raw/` (например, Web Clipper) → ingest по запросу → вопросы в чате → lint периодически. Подробнее: `wiki/LLM Wiki Workflow.md`, правила: `AGENTS.md`.

## Schema для агента

`AGENTS.md` в корне репозитория — единственный источник правил для LLM-агента, который поддерживает vault. Он описывает структуру, жесткие правила (например, неизменяемость `raw/`), workflow ingest/query/lint и обработку lifecycle-вопросов вроде confidence, freshness и конфликтов источников.

Cursor, Codex, Claude Code и многие другие агенты подхватывают `AGENTS.md` автоматически - отдельные промпты или дублирующие rule-файлы для базового поведения не нужны.
