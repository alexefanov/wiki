# Что в этом файле

Этот файл описывает назначение wiki и её целевое состояние - то, как она должна работать.

## Назначение wiki

Это база знаний для людей и LLM (агентов):
- предназначенная для быстрого поиска точной информации.
- содержащая пошаговые инструкции и требования для ключевых процессов.
- опирающаяся на точную информацию из добавленных источников.
- адаптирующаяся к изменениям источников.

## Описание wiki

Wiki состоит из трех слоев:
- Исходные источники - неизменяемые, выбираемые пользователем.
- wiki-страницы - формируемые LLM на основе источников.
- Agents.md и его дочерние страницы - определяющие правила работы и навыки агентов.


# Целевой Workflow пользователя и агента

1. Пользователь сохраняет необработанные источники в папку `/temp`. Эта папка никак не используется.
2. Пользователь запускает команду по обработке необработанного источника. Агент обрабатывает источник и помещает его в папку `raw/`. Обработка включает:
	1. Распознавание сканов и рисунков.
	2. Исправление форматирования, избавление от артефактов
	3. Разбивку на абзацы с заголовками нужного уровня.
	4. Добавление идентификаторов абзацев (с номером пункта или порядковым номером, если номер)

# Разные мысли

1. Необходимо отслеживание разных редакций
2. Нет устойчивых связей по авторитетности источника - он может быть рекомендательным в одном случае и требованием - в другом. 
3. Новая редакция может быть неприменима к старым зданиям. Нужны лишь связи по области применения к документу. А в нем уже ветвление по редакциям.
4. Старый документ не становится менее авторитетным сам по себе - нужно отслеживание противоречий
5. Если пользователь просит проанализировать на дату постройки или проектирования нужно использовать редакции соответствующих лет.
6. Страница документ содержит современные требования. Предыдущие подсвечиваются сносками.
