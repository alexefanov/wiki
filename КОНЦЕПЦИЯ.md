# Что в этом файле

Этот файл описывает назначение wiki и её целевое состояние - то, как она должна работать.

## Назначение wiki

1. Это база знаний для людей и LLM (агентов):
- предназначенная для быстрого поиска точной информации по заданным центральным темам,
- содержащая пошаговые инструкции и требования для ключевых процессов,
- опирающаяся на точную информацию из добавленных источников и граф знаний,
- адаптирующаяся к изменениям источников.

1. Это также полноценная платформа для исследовательского цикла по любой из центральных тем: анализ исходных данных → построение графа знаний по проекту → выявление пробелов → генерация плана реализации → анализ реальных экспериментальных данных → написания итогового отчета → рецензирования и устранения замечаний. Wiki позволяет агенту на основании исходных реальных данных выполнить их обработку и оформление отчета соответствии с этой базой знаний.
## Описание wiki

Wiki состоит из трех слоев:
- Исходные источники - неизменяемые, выбираемые и предоставляемые пользователем.
- wiki-страницы - формируемые LLM на основе источников.
- Agents.md и его дочерние страницы - определяющие правила работы и навыки агентов.

Wiki состоит из двух рабочих пространств:
1. Пространство базы знаний, где происходит накопление информации.
2. Пространство проектов, где информация из базы знаний используется для обработки реальных данных.
Пространство проектов не используется как источник знаний для пространства базы знаний, но используется самим пространством проектов для самообучения при каждой новой задаче. Поэтому в нем хранятся результаты как успешных, так и неуспешных проектов. Каждый проект имеет собственных граф знаний, поскольку может включать в себя элементы разных центральных тем.

# Целевой Workflow пользователя и агента

## Пространство базы знаний
### Инициализация wiki

1. Инициализация (создание) wiki выполняется один раз. Пользователь помещает в выбранную им папку файл `AGENTS.md`, подключает к ней агента и дает команду на инициализацию. 
2. Агент создает в папке базовую структуру подпапок wiki и основные требуемые для ее работы файлы (логи, `index.md` и другие).

### Добавление источников

1. Пользователь сохраняет необработанные источники (в том числе бинарные) в папку `/temp`.
2. Пользователь запускает команду по обработке необработанного источника. Агент обрабатывает источник и помещает его в папку `raw/`. Обработка включает:
	1. Преобразование формул в формат MathJax, а диаграмм - в Mermaid. Если формулы в источнике в виде рисунков, они распознаются, но рисунки тоже сохраняются в тексте для контроля корректности распознавания.
	2. Исправление форматирования, избавление от артефактов (рекламного текста и ссылок), оставляя только текст документа.
	3. Разбивку на абзацы с заголовками нужного уровня.
	4. Оформление нумерованных и маркированных списков.
	5. Замену внешних URL-ссылок на разделы этого же документа на внутренние ссылки.
	6. Сохранение внешних изображений в локальное хранилище для импорта с помощью плагина Local Images Plus.
	7. Добавление идентификаторов абзацев (с номером пункта или порядковым номером, если номера пункта нет).
	8. Добавление содержания по заголовкам абзацев до основного текста.
	9. Добавление свойств - yaml заголовков, содержащих информацию об источнике. Формирование тегов. Обновление `terms.md` на основании имеющихся в источнике терминов и определений. Обновление `biblio.md` на основании нормативных и библиографических ссылок, найденных в тексте документа.
	10. Выявление на основе текста (и веб-поиска) изменений, редакций документа с формированием истории версий.
3. Пользователь проверяет обработанный текст и отмечает его свойство `Проверено` как true. Обработанные источники в `raw/` считаются корректными, неизменяемыми оригиналами исходных документов, если есть отметка пользователя. 
4. По команде пользователя дубликаты обработанных и подтвержденных оригиналов из папки `temp` удаляются.
5. Если пользователь запускает повторную обработку, агент находит обработанный источник в `raw/`, даже если он ранее был создан на основе другого файла в `temp` и вносит нужные изменения. 
6. Разные редакции, переиздания, изменения и т.п. одного документа считаются разными источниками (не дубликатами) и сохраняются отдельно для отслеживания версий.

### Внедрение источников в wiki

1. По команде пользователя производится внедрение (igest) источника в wiki - создание для документа wiki-страницы типа Документ или обновление существующей.
2. Для разных редакций, переизданий, изменений и т.п. одного документа создается (обновляется) только одна страница. На ней всегда приведены саммари и цитаты из последней версии, но все отличия от предыдущих версий приводятся в примечаниях. Приводится история версий с датами вступления в силу каждой. При добавлении новой версии документа, wiki-страница обновляется.


# Разные мысли

1. Необходимо отслеживание разных редакций
2. Нет устойчивых связей по авторитетности источника - он может быть рекомендательным в одном случае и требованием - в другом. 
3. Новая редакция может быть неприменима к старым зданиям. Нужны лишь связи по области применения к документу. А в нем уже ветвление по редакциям.
4. Старый документ не становится менее авторитетным сам по себе - нужно отслеживание противоречий
5. Если пользователь просит проанализировать на дату постройки или проектирования нужно использовать редакции соответствующих лет.
6. Страница документ содержит современные требования. Предыдущие подсвечиваются сносками.
